# Пользовательские сценарии и реализация в TaskSolver

Документ связывает сценарии из пояснительной записки с модулями веб-приложения и API.

---

## Структура интерфейса (панель быстрого доступа)

| Вкладка | Сценарии | Кто видит |
|---------|----------|-----------|
| **Обзор** | Аналитика, загрузка команд, личная загрузка исполнителя | Все |
| **Проекты** | Управление проектами и задачами в проекте | Все видят список; создавать — admin и project_manager |
| **Задачи** | Канбан/таблица, «Мои задачи», редактирование | Все (по правам) |
| **Команды** | Загрузка ресурсов, состав команд | Все |
| **Оптимизация** | Область эффективных решений (A, C, D, F) | admin, project_manager, team_lead |
| **Админ** | Роли и доступ пользователей | Только admin |

Логика прав: `taskSolver.front/src/lib/permissions.ts`, проверки на API — `middleware/auth.js`.

---

## 1. Управление проектами и задачами

### Создание и распределение задач (руководитель проекта)

| Реализация | Путь |
|------------|------|
| Создание проекта | `ProjectsPanel` → `POST /api/projects` |
| Добавление задач в проект | `POST /api/projects/:id/tasks` |
| Назначение команды | поле `assignedTeamId`, канбан, массовое обновление |
| Сроки | `deadline` в форме задачи |

**Роли:** просмотр всех проектов — все роли; создание проектов — `admin`, `project_manager`; создание задач и назначение команды — все, кроме `viewer`.

### Редактирование параметров и приоритетов (аналитик / тимлид)

| Реализация | Путь |
|------------|------|
| Карточка задачи | модальное окно, `PUT /api/tasks/:id` |
| Приоритет 1–3 | `business_priority`, clamp в API |
| Массовое изменение | `POST /api/tasks/bulk-update` |

**Роли:** `canEditTasks` — admin, project_manager, team_lead, developer.

### Список назначенных задач (участник команды)

| Реализация | Путь |
|------------|------|
| Фильтр **«Мои задачи»** | задачи с `assignedTeamId = teamId` пользователя |
| После входа | `GET /api/auth/me` (поле `teamId`, команда) |

**Роли:** прежде всего `developer`; доступен всем с привязкой к команде.

## 2. Анализ загрузки ресурсов

### Загрузка сотрудников / команд (руководитель)

| Реализация | Путь |
|------------|------|
| Графики и KPI | `AnalyticsPanel`, вкладка «Обзор» |
| Загрузка команд | `GET /api/visualization/load-chart`, `GET /api/teams/load` |
| Вкладка «Команды» | прогресс-бары `currentLoad / capacity` |

Формула: \(L_i = T_i / K_i\), отображение в процентах.

### Личная занятость (исполнитель)

| Реализация | Путь |
|------------|------|
| Блок **«Моя загрузка»** | на вкладке «Обзор», если у пользователя задана команда |
| Данные | та же метрика загрузки команды пользователя |

## 3. Построение области эффективных решений

### Несколько вариантов распределения (руководитель / менеджер)

| Реализация | Путь |
|------------|------|
| Запуск анализа | кнопка «Запустить оптимизацию», `GET /api/tasks/optimize` |
| Варианты A, C, D, F | `src/utils/optimizer.js`, см. `docs/optimization.md` |
| Применение решения | `POST /api/tasks/optimize/apply` |

**Роли:** `admin`, `project_manager`, `team_lead` — вкладка «Оптимизация» скрыта у остальных.

### Пересчёт при изменении условий

| Реализация | Путь |
|------------|------|
| Повторный расчёт | кнопка «Пересчитать» на вкладке «Оптимизация» |
| Обновление данных | «Обновить» в шапке, изменение задач/команд → новый прогон |

## 4. Аналитика и визуализация

### Экспорт для отчётности

| Реализация | Путь |
|------------|------|
| Экспорт CSV | кнопка на вкладке «Задачи», `GET /api/tasks/export?format=csv` |
| Содержимое | задачи, приоритет, статус, команда, стоимость |

**Роли:** `canViewReports` — admin, project_manager, team_lead, viewer (не developer).

### Наблюдатель (viewer)

- В UI: канбан без перетаскивания, таблица без массовых действий и inline-редактирования, карточка задачи в режиме «Просмотр».
- API: middleware `denyViewer` на `POST/PUT/DELETE` задач и команд; редактирование проектов запрещено в `canEditProject`.

**Тестовый вход:** `viewer@example.com` / `viewer123`.

## 5. Интерфейс и взаимодействие

### Простой интерфейс

- Нижняя панель вкладок, единый стиль карточек, канбан и таблица задач.
- Контекстные подсказки на KPI (переход в нужный раздел).

### Управление правами (администратор)

| Реализация | Путь |
|------------|------|
| Вкладка **«Админ»** | `UsersAdminPanel` |
| API | `GET/POST/PUT/DELETE /api/users` |
| Действия | создание и редактирование в модальных окнах, роль, команда, блокировка |
